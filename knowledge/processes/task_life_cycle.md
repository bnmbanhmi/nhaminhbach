# Task Lifecycle

tag: #processes 


1.  **Creation:** A new Task file is created from `[[Template - Task]]` when a new work item is identified during Sprint Planning. Status is set to `todo`.
2.  **Execution:** The assigned owner works on the task. Status is changed to `in-progress`. All relevant conversations and decisions are logged within this file.
3.  **Completion:** Once all Acceptance Criteria are met, the status is changed to `done`.
4.  **Capture:** The Knowledge Capture process (`[[Process - Knowledge Capture]]`) is executed on this file.
5.  **Archival:** The task is considered complete.
