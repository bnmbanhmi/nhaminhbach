## üéØ Best Practices for Migration

### **Do:**
- ‚úÖ Preserve existing business context and requirements
- ‚úÖ Keep completed work documentation for reference
- ‚úÖ Update time estimates to reflect AI-accelerated pace
- ‚úÖ Add AI context sections gradually as you use them
- ‚úÖ Focus on essential context that AI agents need

### **Don't:**
- ‚ùå Migrate everything at once - do it incrementally
- ‚ùå Lose valuable retrospective insights from completed work
- ‚ùå Remove business context in favor of pure technical focus
- ‚ùå Overcomplicate - keep templates lean and focused
- ‚ùå Forget to update agent instructions and process files

## ‚ö†Ô∏è Common Migration Issues

### **Issue: Information Overload**
**Problem:** AI agents get confused by too much information
**Solution:** Use "Essential Context Files" section to guide focus

### **Issue: Lost Business Context**
**Problem:** Technical focus loses sight of user needs
**Solution:** Maintain clear user stories and success criteria in epics

### **Issue: Session Discontinuity**
**Problem:** AI agents lose context between sessions
**Solution:** Enforce rigorous "AI Session Log" updates

### **Issue: Escalation Confusion**
**Problem:** Unclear when AI should make decisions vs. escalate
**Solution:** Use clear "AI Decision Framework" in epics and tasks
